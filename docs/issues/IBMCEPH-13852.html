Preflight playbook should not force-enable firewalld (make it configurable/conditional)

Developer Analysis (from JIRA comments):
  • Is there any possible workaround that can be performed before running the preflight playbook as we have another customer hitting this issue? SELinux and FirewallD both got enabled post running the playbook and it impacted the cluster communication
  • As a workaround, you can modify the FirewallD task in cephadm-preflight.yml and set enabled: false before running the preflight playbook. by default, it is true
# cephadm-preflight.yml
  • name: Ensure firewalld is enabled and running
ansible.builtin.systemd:
name: firewalld
state: sta
