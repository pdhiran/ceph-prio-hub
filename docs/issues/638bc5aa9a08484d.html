Notes: Customer-reported issue. Clone IBMCEPH-15734 created for 9.1z. Multiple related SSL/dashboard 500 errors exist across versions suggesting systemic issue with cert handling in dashboard backend.

Analysis

Root Cause Analysis
Dashboard returns 500 Internal Server Error when uploading/viewing RGW SSL certificates. Root Cause: The dashboard SSL certificate management UI makes API calls to the mgr module REST endpoint for certificate operations. The error occurs when the dashboard attempts to parse or validate the SSL certificate PEM file during upload. The mgr dashboard backend fails to handle the cert.pem file, producing a traceback in mgr logs ("Error while calling Task(service/edit_mds/service_name)" pattern seen in logs). Related Issues: - IBMCEPH-15734: Clone for 9.1z stream (same issue, Assigned) - IBMCEPH-13816: 500 error deploying RGW Ingress with SSL via Dashboard (Assigned, v9.0z2) - IBMCEPH-996 (BZ#1954645): "ceph dashboard set-ssl-certificate{,-key} -i" commands fail (Fixed in 5.0) - IBMCEPH-1330 (BZ#1987338): Invalid SSL certificate on creating RGW service with SSL (Fixed in 5.0) Known Issue: Confirmed as known issue in ceph-issue-kb (3 matches). No fix merged yet. Status: ON_QA in JIRA. Affected Version: IBM Ceph 9.0z2
